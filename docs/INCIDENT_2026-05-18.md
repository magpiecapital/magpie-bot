# Incident — 2026-05-18 — Cosign-Borrow Drain

**Status:** Mitigated. No user funds were affected at any point.
**Loss:** 4.304 SOL from the protocol's lender treasury, across 7 transactions in a 4h 28m window.

## What happened

A borrower-side actor crafted a `borrow` transaction that included an
*extra* `SystemProgram.transfer` instruction in the same tx as the legit
`borrow` call. Because the lender authority co-signs `borrow`
transactions to enable one-click flows, that co-sign also authorized the
extra transfer — pulling SOL from the lender's treasury wallet to an
attacker-controlled address. The protocol's user-facing loan flow worked
exactly as designed; the attack abused the co-signing endpoint's lack of
outer-instruction filtering.

**No user collateral, no user wallet, and no LP funds were touched.** The
loss was entirely from the protocol's lender treasury — funds the
operator had deposited to seed lending.

## What was patched (same day)

1. **Outer-program allowlist** in the cosign-borrow endpoint — the
   server now rejects any transaction whose outer instructions reference
   a program outside an explicit allowlist (the Magpie program(s),
   ComputeBudget, ATA, Token program, Token-2022).
2. **Defense-in-depth System-from-lender block** — the endpoint
   explicitly refuses to co-sign any transaction containing a
   `SystemProgram.transfer` whose source is the lender authority.
3. **`COSIGN_BORROW_DISABLED` env kill-switch** — operator can disable
   the entire co-sign endpoint without a redeploy if a future variant
   is discovered.
4. **Helius real-time alarm** webhook on the lender wallet — any
   non-allowlisted outflow now DMs the operator within seconds.

The application-layer patch is live on Railway; the on-chain program
was not redeployed (no authority-transfer instruction exists in the
current IDL, and a redeploy would touch every existing loan). A
key-rotating v3 program is on the longer-term roadmap.

## Audit + verification

A 7-day audit was completed using both RPC and Helius enhanced-API
tools. Total drain confirmed at 4.304 SOL across 7 transactions inside
a 4h 28m window. No earlier or later drains found. Loan and LP flows
during and after the incident were verified normal on-chain.

## What's monitored now

- Real-time alarm on lender-wallet outflows (Helius webhook → operator DM)
- Daily operator digest of cosign endpoint usage anomalies
- All cosign-borrow transactions logged with their outer instructions
  for audit
